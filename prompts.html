ASSIGNMENT DOCUMENTATION · AI PROMPTING PROJECT

THE
PROMPTS

This page documents every AI prompt used to build this site from scratch. No code was written manually — all HTML, CSS, and JavaScript was produced through natural language prompting of Claude (Anthropic).

Project Overview: This is a fan-made anticipation page for a fictional Persona 4 remake announcement. Every file — index.html, style.css, script.js, and this prompts.html — was generated entirely by Claude. No manual coding was performed at any stage. The site was built in two rounds of prompting: an initial generation pass, followed by a full enhancement pass that added immersive fog layers, a TV static intro, flip-digit countdown, parallax tilt, scroll reveal animations, and deeper atmospheric styling.

02
Homepage HTML Structure
index.html
Prompt Used 
Generate the index.html file for the Persona 4 Revival website.

Include:
- A sticky navigation bar with links to Home, Team, Theories, and the Prompts page
- A dramatic hero section with the site title and a call-to-action button
- A countdown timer section with day/hour/minute/second blocks tied to a fictional TGS 2025 reveal date

Use semantic HTML5. Link to style.css and script.js.
What This Resulted In

Claude generated the complete HTML structure including a fixed navbar, full-screen hero section with an animated floating Izanagi card, a countdown grid, six Investigation Team character cards with kanji avatars and arcana labels, a two-column theory submission layout, and a footer. The second generation pass added a TV static intro screen, fog container layers, floating arcana sigils, a scroll hint indicator, affinity badges on character cards, and a character counter on the theory textarea.

03
Visual Design & Styling
style.css
Prompt Used 
Generate the style.css file for the Persona 4 Revival website.

Design requirements:
- Dark blue color palette inspired by Persona 4 (deep navy #050d1a, gold #f0c040 as accent)
- Use CSS custom properties (variables) for the entire color system
- Fonts: Bebas Neue for headings, Rajdhani for body text, Share Tech Mono for labels
- Animated floating Persona card in the hero section
- Countdown blocks with gold top border and monospace numbers
- Character cards with hover lift effects and color-coded kanji avatars per character
- Theory form with dark inputs and gold focus borders
- Responsive layout for mobile screens
- A noise/grain overlay effect for atmosphere
- Smooth scroll, subtle fade-in animations via CSS transitions
What This Resulted In

The first pass produced a comprehensive CSS file with a 12-token color system, three Google Fonts, card float animation, and responsive breakpoints. The enhancement pass added: animated fog layers drifting across the viewport (three stacked layers at different speeds), a TV intro with scanlines, floating arcana sigils with randomized drift animations, a shimmer sweep on the eyebrow tag, staggered title-in animations for each hero headline, flip-digit keyframes on countdown numbers, 3D tilt-ready perspective on character cards with a shine sweep element, a pulsing seconds block, affinity color badges, improved empty state for the theories board, and a footer fog gradient.

04
JavaScript Interactivity
script.js
Prompt Used 
Generate the script.js file for the Persona 4 Revival website.

Include:
- A live countdown timer that targets September 25, 2025 (JST, TGS reveal date)
  - Updates every second, displays days/hours/minutes/seconds
  - Shows a congratulations message when the date passes
- A fan theory submission system:
  - Validates that the theory text is not empty and at least 10 characters
  - Saves theories using sessionStorage (no backend needed)
  - Renders submitted theories as cards, newest first
  - Shows success/error feedback messages
  - Displays a count badge of total theories submitted
- Navbar scroll effect (increases opacity on scroll)
- IntersectionObserver fade-in for character cards and countdown blocks
- Active nav link detection based on current page URL

Use vanilla JavaScript only. No frameworks.
What This Resulted In

The first pass produced a countdown, theory system with sessionStorage, and basic IntersectionObserver fade-ins. The enhancement pass added eight distinct modules: (1) a TV static intro that renders live canvas noise and fades out after ~2.2 seconds; (2) flip-digit animation that only fires when a countdown digit changes; (3) a textarea character counter (0/500) with live updates; (4) a badge pop animation on theory submission; (5) per-card staggered scroll reveal with delay offsets; (6) 3D tilt on character cards that tracks the mouse cursor with rotateX/Y perspective; (7) a hero parallax that moves content, the Persona card, and floating sigils in opposing directions on mouse move; and (8) improved navbar class toggle rather than direct style manipulation.

07
Deployment Guidance
Meta
Prompt Used 
After generating the files, also suggest how I can deploy this as a public website.
What This Resulted In

Claude provided three free hosting options for static multi-file websites: GitHub Pages (create a repo, upload all four files, enable Pages under Settings), Netlify drag-and-drop deployment (upload folder directly to dashboard), and Vercel (connect GitHub repo for auto-deploy on every push). All options support this site without any backend since it uses only sessionStorage.

Final Summary: Seven prompts — covering architecture, HTML, CSS, JavaScript, documentation, enhancement, and deployment — were enough to generate a complete, multi-page, animated, interactive website from scratch using only natural language. No manual code editing was performed at any stage. The final site contains approximately 1,600+ lines of HTML, 800+ lines of CSS, and 300+ lines of JavaScript, all AI-generated.
